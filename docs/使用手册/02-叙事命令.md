# 02 叙事命令

本章逐条说明根命令的语法、参数、前置状态、模型边界、终端输出、JSON 输出、文件写入、退出码和示例。

返回[手册首页](README.md)。

## 1. `doctor`

用途：检查 Python、关键依赖、目录写权限、API Key，并可在线检查 DeepSeek 模型列表。

### 1.1 语法与参数

```powershell
uv run movie-breakdown doctor `
  [--directory PATH] `
  [--online|--no-online] `
  [--json]
```

| 参数 | 类型 | 必填 | 默认值 | 说明 |
| --- | --- | ---: | --- | --- |
| `--directory PATH` | 路径选项 | 否 | 进程启动目录 | 要检查写权限的目录 |
| `--online / --no-online` | 布尔对偶开关 | 否 | `--online` | 是否调用 DeepSeek 模型列表 |
| `--json` | 布尔开关 | 否 | 关闭 | 输出 `DoctorReport` JSON |

### 1.2 输入与检查

按顺序检查：

1. Python；
2. Agno、Pydantic、Typer、pypdf、OpenAI 等关键依赖；
3. `--directory` 的创建和写权限；
4. `DEEPSEEK_API_KEY` 是否存在；
5. DeepSeek 模型列表，仅 `--online`。

目标目录不存在时，诊断可以创建目录并用临时文件检查写权限；测试临时文件会清理。`--no-online` 只跳过第 5 项，缺少 Key 仍判为失败。

### 1.3 输出、副作用与退出码

终端模式逐项显示 `PASS`、`FAIL` 或说明。JSON 外层：

```json
{
  "schema_version": "1.0",
  "ok": true,
  "checks": [
    {
      "name": "<检查名称>",
      "status": "<状态>",
      "message": "<说明>"
    }
  ]
}
```

- 全部必要检查通过：退出 0；
- 任一必要检查失败：退出 1；
- 参数用法错误：退出 2。

`doctor` 不创建业务项目。配置对象本身初始化失败时可能在错误包装之前终止，即使带 `--json` 也未必得到 JSON。

### 1.4 示例

```powershell
# 完整在线诊断
uv run movie-breakdown doctor

# 跳过模型列表，但仍检查 Key
uv run movie-breakdown doctor --no-online

# 检查目标目录并输出 JSON
uv run movie-breakdown doctor `
  --directory ".\workspace" `
  --no-online `
  --json
```

## 2. `analyze`

用途：从源剧本创建新项目并执行完整叙事拆解。

### 2.1 语法与参数

```powershell
uv run movie-breakdown analyze SOURCE `
  --project PROJECT `
  [--framework three-act] `
  [--format-detection auto|local|model] `
  [--json]
```

| 参数 | 类型 | 必填 | 默认值 | 说明 |
| --- | --- | ---: | --- | --- |
| `SOURCE` | 路径参数 | 是 | 无 | `.txt`、`.md` 或文本型 `.pdf` |
| `--project PROJECT`、`-p PROJECT` | 路径选项 | 是 | 无 | 要创建的新项目目录 |
| `--framework` | 枚举 | 否 | `three-act` | 当前唯一开放的结构框架 |
| `--format-detection` | 枚举 | 否 | `auto` | 场景格式识别策略 |
| `--json` | 布尔开关 | 否 | 关闭 | 机器可读终端输出 |

### 2.2 输入和前置条件

- `SOURCE` 存在、是普通文件、扩展名受支持且可提取至少 20 个可见字符；
- `PROJECT` 尚未存在 `project.json`；
- 项目父目录可写；
- `DEEPSEEK_API_KEY` 已配置。

缺少 Key 时会在项目初始化前失败，不写项目。项目已经初始化时不要再次 `analyze`，应使用 `resume`。

### 2.3 阶段和输出

九个阶段：

| 阶段 | 主要结果 |
| --- | --- |
| `normalize` | 规范化文本、来源和行/页映射 |
| `scenes` | 确定性场景切分 |
| `scene_analysis` | 逐场叙事分析 |
| `global_analysis` | 实体、事件、关系和全局结构 |
| `character_dossiers` | 全人物分级档案 |
| `character_biographies` | 核心人物声明级小传 |
| `validation` | 一致性和证据校验 |
| `manual_corrections` | 当前人工修正层，初始为空 |
| `export` | 正式 JSON 和 Markdown |

主要写入：

- `project.json`；
- `source/<原文件名>`；
- `artifacts/manifest.json`；
- `artifacts/*.json` 和逐项 `.jsonl`；
- `exports/breakdown.json`；
- `exports/report.md`。

非 JSON 模式把模型进度写到标准错误，完成后显示校验摘要和导出路径。

JSON 成功外层：

```json
{
  "ok": true,
  "project": "<项目绝对路径>",
  "validation": {
    "valid": true,
    "scene_count": 0,
    "analyzed_scene_count": 0,
    "coverage": 0.0,
    "issues": []
  },
  "exports": {
    "json": "<breakdown.json 绝对路径>",
    "markdown": "<report.md 绝对路径>"
  }
}
```

上述数字只是字段形状示意，不是任何项目的实际结果。

### 2.4 退出码

- 全流水线和最终导出成功：0；
- 文件、配置、模型、Schema 或流水线校验异常：1；
- 缺参、无效枚举等用法错误：2。

### 2.5 示例

```powershell
uv run movie-breakdown analyze ".\inputs\sample-screenplay.txt" `
  --project ".\workspace\sample-project" `
  --framework three-act `
  --format-detection auto
```

## 3. `resume`

用途：从失败、未完成、缺失或过期阶段继续叙事分析。

### 3.1 语法与参数

```powershell
uv run movie-breakdown resume PROJECT [--json]
```

| 参数 | 类型 | 必填 | 默认值 | 说明 |
| --- | --- | ---: | --- | --- |
| `PROJECT` | 路径参数 | 是 | 无 | 已初始化的主项目目录 |
| `--json` | 布尔开关 | 否 | 关闭 | 输出与 `analyze` 相同的 JSON 外层 |

### 3.2 输入和行为

需要：

- `project.json`；
- `artifacts/manifest.json`；
- 项目内源文件；
- 可严格读取的现有恢复产物；
- API Key。

框架、格式识别、模型和关键参数从 `project.json` 读取，`resume` 不能临时修改。

恢复规则：

- 复用缓存键、阶段版本和产物指纹全部匹配的成功内容；
- 重跑 `pending`、`failed`、`stale` 或缺失内容；
- 逐场叙事和核心人物小传按单项恢复；
- 刷新受影响的下游阶段；
- 成功后默认重新导出 JSON 和 Markdown。

### 3.3 输出、写入和退出码

输出形状与 `analyze` 相同。`resume` 会更新运行清单、恢复记录、受影响的结构化产物和双格式导出。

- 成功：0；
- 项目、密钥、模型、恢复或校验错误：1；
- 用法错误：2。

### 3.4 示例

```powershell
uv run movie-breakdown status ".\workspace\sample-project"
uv run movie-breakdown resume ".\workspace\sample-project" --json
```

## 4. `status`

用途：查看叙事阶段状态、版本、缓存、Token 用量和失败原因。

### 4.1 语法与参数

```powershell
uv run movie-breakdown status PROJECT [--json]
```

| 参数 | 必填 | 说明 |
| --- | ---: | --- |
| `PROJECT` | 是 | 包含 `artifacts/manifest.json` 的项目 |
| `--json` | 否 | 输出原始 `RunManifest` |

### 4.2 输入与输出

不读取 API Key，不调用模型。终端表格主要列出：

- 阶段；
- 状态；
- 阶段版本；
- Token；
- 失败或过期原因。

JSON 是完整运行清单；以下只节选常用字段，实际 `StageRecord` 还包含名称、版本和起止时间：

```json
{
  "schema_version": "1.0",
  "project_id": "<项目 ID>",
  "stages": {
    "<阶段名>": {
      "status": "success",
      "cache_key": "<缓存键或 null>",
      "artifact_fingerprint": "<产物指纹或 null>",
      "error": null,
      "usage": {
        "input_tokens": 0,
        "output_tokens": 0,
        "total_tokens": 0
      }
    }
  },
  "updated_at": "<UTC 时间>"
}
```

### 4.3 副作用与退出码

通常只读取清单；打开旧版本项目时，如果当前程序新增了阶段，命令会补齐阶段并保存 `artifacts/manifest.json`。

- 成功：0；
- 清单缺失或损坏：1；
- 用法错误：2。

### 4.4 示例

```powershell
uv run movie-breakdown status ".\workspace\sample-project"
uv run movie-breakdown status ".\workspace\sample-project" --json
```

## 5. `validate`

用途：不调用模型地重建当前正式叙事聚合，并重新执行一致性校验。

### 5.1 语法与参数

```powershell
uv run movie-breakdown validate PROJECT [--json]
```

| 参数 | 必填 | 说明 |
| --- | ---: | --- |
| `PROJECT` | 是 | 具有完整上游叙事产物的项目 |
| `--json` | 否 | 输出 `ValidationReport` 或被捕获的错误 JSON |

### 5.2 输入和校验

读取全部叙事上游产物，并重放当前激活的人工修正。主要检查：

- 严格 Pydantic Schema；
- 场景覆盖；
- 场景、人物、事件和关系 ID；
- 行号、页码、证据摘录和引用；
- 全人物档案与核心人物小传；
- 幕、节拍、情节线和正式分析内容指纹。

结构正确不等于叙事判断绝对正确。主题、人物弧光和转折点仍需要 `review` 的人工判断。

### 5.3 输出、写入和退出码

终端显示校验结论、场景覆盖和问题。JSON 报告主要字段：

```json
{
  "valid": true,
  "scene_count": 0,
  "analyzed_scene_count": 0,
  "coverage": 0.0,
  "issues": []
}
```

命令不是零写入：它会刷新 `artifacts/validation.json`、`artifacts/manual_corrections.json`、正式聚合缓存和 `artifacts/manifest.json`；但不会调用模型或生成正式导出。

退出行为：

- 有效：0；
- 如果命令取得 `valid=false` 报告：输出报告后返回 2；
- 当前实现中，上游重建发现无效内容时通常先抛流水线错误，转成应用错误并返回 1；
- 文件或 Schema 错误：1；
- 用法错误：2。

### 5.4 示例

```powershell
uv run movie-breakdown validate ".\workspace\sample-project"
uv run movie-breakdown validate ".\workspace\sample-project" --json
```

## 6. `export`

用途：从当前正式叙事聚合重新生成指定格式的报告，不调用模型。

### 6.1 语法与参数

```powershell
uv run movie-breakdown export PROJECT `
  [--format markdown|json|all] `
  [--json]
```

| 参数 | 类型 | 必填 | 默认值 | 说明 |
| --- | --- | ---: | --- | --- |
| `PROJECT` | 路径参数 | 是 | 无 | 已有叙事项目 |
| `--format` | 枚举 | 否 | `markdown` | 要生成的正式报告格式 |
| `--json` | 布尔开关 | 否 | 关闭 | 在终端输出路径映射 |

### 6.2 输入、输出和写入

命令读取全部必要上游产物，重放当前激活修正，并在导出前重新校验。

| `--format` | 本次写入 |
| --- | --- |
| `markdown` | `exports/report.md` |
| `json` | `exports/breakdown.json` |
| `all` | 同时写上述两个文件 |

先前存在但本次未选择的其他格式不会被删除。

非 JSON 模式逐项显示格式和绝对路径。JSON：

```json
{
  "ok": true,
  "exports": {
    "markdown": "<绝对路径>",
    "json": "<绝对路径>"
  }
}
```

`exports` 只包含本次选择的格式。

### 6.3 退出码与示例

- 成功：0；
- 缺失、校验、读取或写入错误：1；
- 用法错误：2。

```powershell
uv run movie-breakdown export ".\workspace\sample-project" --format all
uv run movie-breakdown export ".\workspace\sample-project" --format json --json
```

## 7. `review`

用途：纯本地生成叙事风险信号、稳定抽样目标和答案模板，也可导入已填写答案。

### 7.1 语法与参数

```powershell
uv run movie-breakdown review PROJECT `
  [--sample-size 6..50] `
  [--answers PATH] `
  [--json]
```

| 参数 | 类型 | 必填 | 默认值 | 说明 |
| --- | --- | ---: | --- | --- |
| `PROJECT` | 路径参数 | 是 | 无 | 当前正式叙事聚合可重建且有效的项目 |
| `--sample-size` | 整数选项 | 否 | `16` | 期望/上限目标数，允许 6—50 |
| `--answers PATH` | 路径选项 | 否 | 无 | UTF-8 严格答案 JSON，允许部分 responses |
| `--json` | 布尔开关 | 否 | 关闭 | 输出自动信号、摘要和路径 |

候选不足时，实际 `target_count` 可以小于 `sample-size`。

### 7.2 不带答案

命令会：

- 确定性计算自动风险代理信号；
- 对当前分析指纹生成稳定目标；
- 把所有响应置为 `unreviewed`；
- 写 `artifacts/semantic_quality.json`；
- 写 `exports/semantic-quality.md`；
- 首次写 `exports/human-review-<分析指纹前12位>.json`。

同名答案模板已经存在时不会覆盖；但正式 `semantic_quality.json` 会更新为本次空白复核状态。因此，不要在已经导入完整答案后无意运行一次不带 `--answers` 的 `review`。

### 7.3 带答案

答案必须：

- 使用当前 `analysis_fingerprint`；
- 使用当前评分标准版本，当前为 `1.1`；
- 只引用当前抽样的 `target_id`；
- 不重复目标或维度；
- 只使用该目标适用的评分维度；
- 不包含未知字段。

允许只提交部分 `responses`；其余目标自动保留为 `unreviewed`。重新导入时应保持生成目标时的 `--sample-size`，否则目标集可能改变。

### 7.4 输出、写入和退出码

终端显示自动信号免责声明、已评数和目标数。JSON 外层：

```json
{
  "ok": true,
  "analysis_fingerprint": "<当前分析指纹>",
  "automatic_signals": [],
  "human_summary": {
    "reviewed_count": 0,
    "target_count": 0,
    "coverage": 0.0,
    "verdict_counts": {},
    "dimension_averages": {},
    "flagged_target_ids": []
  },
  "exports": {
    "artifact": "<semantic_quality.json 绝对路径>",
    "markdown": "<semantic-quality.md 绝对路径>",
    "answers_template": "<答案模板绝对路径>"
  }
}
```

完整目标、上下文和响应看写入文件，终端 JSON 不包含完整 `human_review`。

- 生成或导入成功，即使评审未完成：0；
- 答案过期、Schema 错误或项目无效：1；
- `sample-size` 越界或用法错误：2。

### 7.5 示例

```powershell
# 生成风险抽检和答案模板
uv run movie-breakdown review ".\workspace\sample-project" --sample-size 16

# 导入已填写答案
uv run movie-breakdown review ".\workspace\sample-project" `
  --sample-size 16 `
  --answers ".\review\narrative-answers.json"
```

答案结构详见[04 专家评审与人工修正](04-专家评审与人工修正.md)。

## 8. `correct`

用途：把结构化修正与提出修正的专家答案绑定，先预演或原子激活叙事修正。

### 8.1 语法与参数

```powershell
uv run movie-breakdown correct PROJECT CORRECTIONS `
  --answers ANSWERS `
  [--dry-run] `
  [--json]
```

| 参数 | 类型 | 必填 | 默认值 | 说明 |
| --- | --- | ---: | --- | --- |
| `PROJECT` | 路径参数 | 是 | 无 | 已完成叙事分析的项目 |
| `CORRECTIONS` | 路径参数 | 是 | 无 | UTF-8 严格 `CorrectionSet` JSON |
| `--answers ANSWERS` | 路径选项 | 是 | 无 | 产生本次修正的答案 JSON |
| `--dry-run` | 布尔开关 | 否 | 关闭 | 完整内存预检且零写入 |
| `--json` | 布尔开关 | 否 | 关闭 | 输出分析指纹和回执 |

### 8.2 输入绑定和校验

修正集合必须匹配：

- 当前来源指纹；
- 修正前基础分析指纹；
- 评分标准版本；
- 专家答案规范内容指纹；
- 评审人；
- 被修改对象当前旧值指纹。

每条修正必须引用一条结论为 `partially_supported`、`unsupported` 或 `uncertain`，且带非空 `proposed_correction` 的答案。`review_target_id`、`field`、`object_id`、旧值、replacement 类型和证据都必须有效；证据摘录必须与对应全局行号逐字一致。

### 8.3 可修正字段

| 类别 | `field` |
| --- | --- |
| 场景和事件 | `scene_summary`、`event_summary` |
| 幕和节拍 | `act_summary`、`act_turning_point`、`beat_summary` |
| 情节和伏笔 | `plot_summary`、`foreshadow_description` |
| 人物弧光 | `arc_initial_state`、`arc_desire`、`arc_need`、`arc_final_state` |
| 人物关系 | `relation_development` |
| 人物小传 | `biography_summary`、`biography_claim_statement`、`biography_claim_rationale` |
| 主题和母题 | `theme`、`motif` |

### 8.4 预演和正式写入

`--dry-run`：

- 在内存执行全部指纹、引用、证据、类型和最终结构校验；
- 不改变项目任何字节；
- 返回 `exports: {}`。

正式激活写入：

- `corrections/active.json`；
- `corrections/review_answers.json`；
- `artifacts/manual_corrections.json`；
- `artifacts/corrected_breakdown.json`；
- `artifacts/correction_receipt.json`；
- 刷新的校验和双格式正式导出。

JSON 外层：

```json
{
  "ok": true,
  "dry_run": true,
  "analysis_fingerprint": "<预演结果或已激活结果的分析指纹>",
  "receipt": {
    "applied_correction_ids": ["<修正 ID>"],
    "applied_count": 1
  },
  "exports": {}
}
```

### 8.5 退出码与示例

- 预演或激活成功：0；
- Schema、绑定、指纹、证据、对象或最终校验错误：1；
- 缺少 `--answers` 等用法错误：2。

```powershell
# 零写入预演
uv run movie-breakdown correct ".\workspace\sample-project" `
  ".\review\narrative-corrections.json" `
  --answers ".\review\narrative-answers.json" `
  --dry-run

# 确认预演成功后激活
uv run movie-breakdown correct ".\workspace\sample-project" `
  ".\review\narrative-corrections.json" `
  --answers ".\review\narrative-answers.json"
```

激活后 `analysis_fingerprint` 改变，旧答案立即过期，必须重新 `review`。

## 9. `finalize`

用途：对当前正式分析和当前专家复核执行叙事稳定版门禁。

### 9.1 语法与参数

```powershell
uv run movie-breakdown finalize PROJECT [--json]
```

| 参数 | 必填 | 说明 |
| --- | ---: | --- |
| `PROJECT` | 是 | 已有当前正式质量报告的项目 |
| `--json` | 否 | 输出完整门禁报告和路径 |

### 9.2 输入和八项门禁

命令读取当前正式拆解和 `artifacts/semantic_quality.json`，检查：

1. 确定性结构校验通过；
2. 质量报告绑定当前分析指纹；
3. `reviewer` 非空；
4. 实际抽检目标至少 16 个；
5. 所有目标 100% 完成；
6. 不存在 `unreviewed` 或 `unsupported`；
7. 每个目标的全部适用维度恰好评分一次，且分数为 1—5；
8. `partially_supported` 或 `uncertain` 的 `notes` 以精确前缀 `接受风险：` 开始，并包含具体说明。

### 9.3 输出、写入和退出码

无论通过还是阻断，都写：

- `artifacts/release_gate.json`；
- `exports/release-gate.md`。

非 JSON 模式逐项显示通过或阻断。JSON：

```json
{
  "ok": false,
  "report": {
    "stable": false,
    "analysis_fingerprint": "<当前分析指纹>",
    "checks": []
  },
  "exports": {
    "artifact": "<release_gate.json 绝对路径>",
    "markdown": "<release-gate.md 绝对路径>"
  }
}
```

- 全部门禁通过：0；
- 逻辑门禁阻断：保存报告后返回 2；
- 质量报告缺失、过期、损坏或项目错误：1；
- 用法错误：2。

### 9.4 示例

```powershell
uv run movie-breakdown finalize ".\workspace\sample-project"
uv run movie-breakdown finalize ".\workspace\sample-project" --json
```
