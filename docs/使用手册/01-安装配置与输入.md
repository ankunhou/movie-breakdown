# 01 安装配置与输入

本章说明运行环境、DeepSeek 配置、剧本输入限制、项目生命周期、模型边界、机器输出和退出码。

返回[手册首页](README.md)。

## 1. 命令约定

本文默认在仓库根目录使用 PowerShell：

```powershell
uv run movie-breakdown <命令> [参数]
```

如果程序已经安装到 `PATH`，可直接使用：

```powershell
movie-breakdown <命令> [参数]
```

示例路径：

| 用途 | 通用示例 |
| --- | --- |
| 输入剧本 | `.\inputs\sample-screenplay.txt` |
| 主项目 | `.\workspace\sample-project` |
| 专家答案 | `.\review\answers.json` |
| 修正集合 | `.\review\corrections.json` |

PowerShell 多行命令使用反引号续写，反引号后不能有空格。

## 2. 系统要求与安装

- Python 3.12 或更高版本；
- uv；
- 受支持的文本输入；
- 对工作目录的读写权限；
- 模型命令所需的 DeepSeek API Key 和网络。

安装并检查版本：

```powershell
uv sync
uv run movie-breakdown --version
```

`--version` 成功时只向标准输出写版本号并返回 0。根命令不带任何子命令时会显示帮助，但退出码为 2；显式 `--help` 返回 0。

## 3. 环境变量

复制模板：

```powershell
Copy-Item ".env.example" ".env"
```

完整配置：

```dotenv
DEEPSEEK_API_KEY=
MOVIE_BREAKDOWN_MODEL=deepseek-v4-pro
MOVIE_BREAKDOWN_MAX_RETRIES=2
MOVIE_BREAKDOWN_CONCURRENCY=4
MOVIE_BREAKDOWN_THINKING_ENABLED=true
MOVIE_BREAKDOWN_REASONING_EFFORT=high
MOVIE_BREAKDOWN_REQUEST_TIMEOUT_SECONDS=600
```

| 环境变量 | 默认值 | 约束 | 作用 |
| --- | --- | --- | --- |
| `DEEPSEEK_API_KEY` | 无 | 应填写真实非空密钥 | 模型命令和 `doctor` 的密钥检查 |
| `MOVIE_BREAKDOWN_MODEL` | `deepseek-v4-pro` | 模型标识 | 固化到项目配置并进入缓存身份 |
| `MOVIE_BREAKDOWN_MAX_RETRIES` | `2` | 0—5 | 每次模型调用的额外重试次数 |
| `MOVIE_BREAKDOWN_CONCURRENCY` | `4` | 1—32 | 并发模型任务数 |
| `MOVIE_BREAKDOWN_THINKING_ENABLED` | `true` | 布尔值 | 是否启用思考模式 |
| `MOVIE_BREAKDOWN_REASONING_EFFORT` | `high` | `high`、`max` | 推理强度 |
| `MOVIE_BREAKDOWN_REQUEST_TIMEOUT_SECONDS` | `600` | 大于 0 | 单次请求超时秒数 |

配置读取规则：

1. 从命令运行时的当前工作目录读取 UTF-8 `.env`。
2. 系统环境变量优先于 `.env`。
3. 首次 `analyze` 会把模型、思考、重试、并发等结果相关配置写入 `project.json`。
4. 后续修改 `.env` 不会静默改写既有项目配置。
5. API Key 不写入项目、缓存、导出或日志。

配置值本身不合法时，错误可能在工作流错误包装之前发生；即使带 `--json`，也不保证得到 JSON。

当前本地 Key 检查只判断该变量是否被配置为非 `None`；空字符串仍可能通过本地存在性检查，但不能完成在线模型请求。不要把空值通过诊断理解成密钥可用，使用默认在线 `doctor` 验证实际连接。

## 4. 剧本输入

### 4.1 支持格式

| 文件 | 支持情况 | 读取方式 |
| --- | --- | --- |
| `.txt` | 支持 | UTF-8 BOM → UTF-8 → GB18030 |
| `.md` | 支持 | 与 TXT 相同 |
| `.pdf` | 支持 | 直接提取文本并保留页码映射 |
| 扫描 PDF | 不直接支持 | 先用外部 OCR 转成文本型 PDF 或 TXT |
| `.fdx`、`.docx`、`.markdown` | 不支持 | 先转换 |

源文件必须：

- 存在且是普通文件；
- 扩展名受支持；
- 能提取至少 20 个可见字符；
- 在项目初始化时可读取。

`analyze` 会把源文件复制到 `PROJECT/source/<原文件名>`，之后恢复和指纹核对以项目内副本为准。文本标题取首个非空行并移除 Markdown 标题符；PDF 标题取文件名。

### 4.2 格式识别策略

| `--format-detection` | 行为 |
| --- | --- |
| `auto` | 本地切分可信则直接使用，否则调用模型识别格式 |
| `local` | 只使用内置中英文规则，切分不可信即失败 |
| `model` | 强制模型生成格式画像和候选正则 |

模型辅助识别只发送有限样本：

- PDF：前三页和后三页；
- TXT、Markdown：首尾各约 150 行。

候选正则还必须通过本地安全和质量检查，包括行首锚定、长度上限、危险语法拒绝、标题样例覆盖、场景数量及平均场长检查。模型生成的正则不会未经验证直接执行。

`local` 只影响格式识别；后续逐场和全局叙事分析仍会访问模型。

## 5. 项目状态与清单

主项目由 `analyze` 初始化；同一目录之后使用 `resume`。主叙事运行清单位于：

```text
PROJECT/artifacts/manifest.json
```

制片作用域由 `production analyze` 初始化，运行清单位于：

```text
PROJECT/production/manifest.json
```

阶段状态：

| 状态 | 含义 |
| --- | --- |
| `pending` | 尚未运行 |
| `running` | 最近记录为执行中，进程可能非正常退出 |
| `success` | 当前缓存身份下成功 |
| `failed` | 已记录失败 |
| `stale` | 上游、版本、Prompt、Schema 或配置变化导致过期 |

`resume` 只重跑失败、未完成、缺失或过期内容。逐场叙事、逐场制片和人物小传按独立 JSONL 记录恢复。

## 6. 模型与本地命令边界

| 命令 | 读取 Key | 访问 DeepSeek |
| --- | ---: | ---: |
| `doctor` | 是 | 默认访问模型列表；`--no-online` 时不访问 |
| `analyze`、`resume` | 是 | 是 |
| `production analyze`、`production resume` | 是 | 是 |
| 其他叙事和制片命令 | 否 | 否 |

模型命令即使所有业务阶段最终命中缓存，也会在构造模型适配器前要求 API Key。

模型命令会发送完成对应分析所需的剧本文本。敏感项目应先确认组织对第三方模型、跨境传输、日志和保密材料的政策。

## 7. `--json` 与文件导出

`--json` 表示终端机器可读输出，而不是生成 JSON 报告文件：

```powershell
# 在终端输出 JSON
uv run movie-breakdown status ".\workspace\sample-project" --json

# 生成 exports/breakdown.json
uv run movie-breakdown export ".\workspace\sample-project" --format json
```

成功结果和进入命令后被捕获的应用错误会输出一个 JSON 对象。模型进度在 JSON 模式下关闭；非 JSON 模式的模型进度写到标准错误。

Typer 参数缺失、非法枚举、`--sample-size` 越界等错误发生在命令函数之前，即使指定 `--json` 也会输出用法文本，而不是 JSON。

捕获的应用错误：

```json
{
  "ok": false,
  "error": "异常类型: 错误说明"
}
```

脚本必须同时检查退出码与 JSON 解析结果。

## 8. 退出码

| 退出码 | 通用含义 |
| ---: | --- |
| `0` | 命令按自身成功契约完成；更高准备度仍应查看输出字段 |
| `1` | 配置、文件、Schema、网络、模型或应用运行错误 |
| `2` | Typer 用法错误，或明确返回的校验/封版门禁阻断 |

注意：

- `review` 即使答案未填完也可以返回 0；
- `production plan` 只要 `draft_valid=true` 就返回 0，即使 `catalog_ready` 或 `shoot_ready` 为假；
- `finalize` 门禁阻断会先保存报告，再返回 2；
- 当前叙事 `validate` 在重建上游时发现无效内容，通常先转成应用错误并返回 1；只有命令实际拿到 `valid=false` 报告时才走退出码 2；
- `production validate` 拿到 `valid=false` 报告时会输出报告并返回 2。

PowerShell：

```powershell
uv run movie-breakdown validate ".\workspace\sample-project" --json
if ($LASTEXITCODE -ne 0) {
  throw "校验未成功，退出码：$LASTEXITCODE"
}
```
