# 社区行为准则

## 我们的承诺

我们致力于为所有参与者提供友善、安全、专业且不受骚扰的协作环境，不因年龄、身体特征、残障、族群、性别身份或表达、经验水平、国籍、教育背景、外貌、种族、宗教、性取向或社会经济状况而区别对待。

## 期望行为

- 以尊重、清晰和有建设性的方式交流；
- 对不同经验、创作方法和专业判断保持开放；
- 聚焦事实、代码、接口和可验证的行为，而非攻击个人；
- 接受合理反馈，承认错误并主动修正；
- 尊重版权、保密义务、个人隐私及剧组安全边界；
- 在发现敏感材料或凭据时停止传播，并依照安全政策私密报告。

## 不可接受的行为

- 侮辱、威胁、骚扰、歧视、跟踪或未经欢迎的性关注；
- 人身攻击、恶意揣测、持续打断讨论或煽动他人骚扰；
- 未经许可公开他人的个人信息、私密沟通或身份信息；
- 上传真实、保密、未授权剧本，泄露凭据或传播敏感日志；
- 利用项目讨论推广违法、危险或规避现实安全审批的做法；
- 维护者认定会损害社区安全和协作秩序的其他行为。

## 适用范围

本准则适用于仓库、Issue、Pull Request、代码评审、讨论区、发布渠道，以及代表本项目参与公开活动的场合。

## 报告与执行

一般分歧可在原讨论中请求维护者介入。若报告涉及隐私、骚扰或其他不宜公开的信息，请发送邮件至 [houankun@hotmail.com](mailto:houankun@hotmail.com)，主题以 `[CODE OF CONDUCT]` 开头；不要在公开 Issue 中披露细节。若事件同时包含漏洞、凭据或敏感材料泄露，请优先按照 [安全政策](SECURITY.md) 使用 GitHub Private Vulnerability Reporting；标题可注明“行为准则”，以便维护者正确分流。

如果暂时无法建立私密联系，可以创建一个不含人员身份和事件细节的 Issue，请求维护者建立私密沟通渠道。

维护者会在尊重报告者安全与隐私的前提下调查，并可根据严重程度采取提醒、要求纠正、临时限制参与、永久禁止参与或向平台报告等措施。恶意或明知不实的举报本身也可能违反本准则。维护者应回避与自己存在直接利益冲突的案件。

## 责任边界

维护者负责解释并执行本准则，但不会公开讨论保密调查的细节。项目社区规则不替代 GitHub 平台条款、当地法律或现实工作场所的正式申诉程序。
