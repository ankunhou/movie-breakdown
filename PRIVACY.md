# 隐私与数据处理说明

本说明描述当前版本在本地和 DeepSeek API 之间的数据流，不构成对 DeepSeek、操作系统、云同步软件或其他第三方的隐私承诺。处理未公开剧本前，请先理解这些边界。

## 哪些命令会联网

以下命令会调用 DeepSeek，并向其发送完成对应分析所需的剧本文本、结构化上下文或派生信息：

| 命令 | 可能发送的内容 |
| --- | --- |
| `analyze` | 格式识别所需的首尾样本、逐场文本，以及全局叙事和核心人物小传所需的场景证据与派生上下文 |
| `resume` | 失败、缺失或过期阶段重新执行所需的上述内容 |
| `production analyze` | 逐场制片元素拆解所需的场景文本和相关场景信息 |
| `production resume` | 失败、缺失或过期制片阶段重新执行所需的上述内容 |

模型请求在结构化校验失败时可能重试，因此同一段或相关内容可能被发送多次。`--format-detection local` 只表示场景格式识别使用本地规则；完整叙事分析的后续阶段仍会调用 DeepSeek。

`doctor` 默认或使用 `--online` 时会调用 DeepSeek 的模型列表接口，用于检查账户可访问的模型；该命令不会读取项目剧本。`doctor --no-online` 不发起该在线检查。

显式启用的 `live` 测试也会访问 DeepSeek，并可能产生费用。测试输入只能使用合成、公版或已获得明确授权的材料。

## 哪些命令纯本地运行

下列命令不构造模型客户端，也不会为了执行自身功能向 DeepSeek 发送数据：

- `--version` 和命令帮助；
- `doctor --no-online`；
- 叙事命令：`status`、`validate`、`export`、`review`、`correct`、`finalize`；
- 制片命令：`production status`、`production validate`、`production export`、`production plan`、`production review`、`production correct`、`production finalize`。

“纯本地”只描述本项目这些命令的实现。操作系统、终端、编辑器、云盘、杀毒软件或其他同时运行的软件是否收集或同步文件，不受本项目控制。

## API Key

`DEEPSEEK_API_KEY` 从进程环境或当前工作目录的 `.env` 环境配置文件读取。项目使用 Pydantic `SecretStr` 承载密钥，持久化的 `project.json` 只包含模型名称、重试、并发和分析策略等非密钥配置；密钥不会写入项目产物、分析缓存或导出文件。

不要把 `.env`、终端环境转储、错误截图或包含凭据的日志提交到 Git。若密钥可能泄露，应立即在 DeepSeek 侧撤销并轮换，再按照 [安全政策](SECURITY.md) 私密报告由本项目造成的泄露。

## Agno 遥测

当前实现为每次结构化调用创建无记忆、无工具的 Agno Agent，并显式设置 `telemetry=False`。这会关闭该 Agent 的 Agno 遥测，但不会阻止完成分析所必需的 DeepSeek API 请求，也不代表 DeepSeek 不会按照其自身条款处理请求、账户和使用信息。

## 本地文件与缓存

`analyze --project <目录>` 会在用户指定的项目目录中保存：

- `source/`：源剧本的本地副本；
- `artifacts/`：规范化文本、场景、模型结果、内容指纹、运行清单、恢复记录和叙事闭环产物；
- `exports/`：用户生成的 JSON、Markdown 等叙事导出；
- `production/`：制片分析、规划、评审、修正、封版和制片导出。

当前实现用于恢复和复用的剧本分析缓存也位于该项目目录及其清单、JSON/JSONL 产物中，而不是由 CLI 自动上传到一个独立的项目云端缓存。产物和导出可能包含大量原文、人物信息与制作判断，应当按照源剧本同等或更高的敏感等级保护。

用户负责：

- 选择访问权限合适、不会被意外公开或同步的项目目录；
- 管理本地副本、备份、版本控制和云盘同步范围；
- 在不再需要时删除整个项目目录、原始输入和自行创建的副本；
- 根据自身合规要求选择普通删除、加密介质或安全擦除方式。

删除本地项目目录不会撤回已经发送给 DeepSeek 的请求，也不会删除第三方可能依据其条款保留的数据。文件系统快照、备份、云同步历史和固态存储特性也可能使普通删除无法等同于不可恢复擦除。

## 用户责任与第三方服务

使用者必须确认自己有权读取、复制、分析并向 DeepSeek 发送所选剧本及相关资料，包括满足版权许可、保密协议、个人信息保护、组织政策和数据跨境要求。

在运行任何模型命令前，应自行审阅当前适用的 DeepSeek 服务条款、隐私政策、数据保留规则、部署地区和账户设置。本项目无法代表 DeepSeek 作出保密、删除、训练用途、数据驻留或服务可用性承诺。

如果剧本不允许交给第三方模型服务，请不要运行 `analyze`、`resume`、`production analyze` 或 `production resume`。当前版本不提供完整叙事或制片分析的纯本地模型替代方案。

## 问题反馈

隐私相关的一般建议可以提交不含敏感信息的 GitHub Issue。涉及漏洞、密钥、真实剧本或敏感数据泄露时，请按照 [安全政策](SECURITY.md) 优先使用 GitHub Private Vulnerability Reporting。
