# 支持说明

## 获取帮助

在提问前，请先阅读 [README](README.md) 和 [完整 CLI 使用手册](docs/使用手册/README.md)，并运行本地诊断：

```powershell
uv run movie-breakdown doctor --no-online
uv run movie-breakdown --version
```

没有找到答案时，可以通过 GitHub Issues 提交：

- 可重复的软件缺陷；
- 文档错误；
- 功能建议；
- 不含敏感内容的使用问题。

Issue 最好包含项目版本、Python 版本、操作系统、执行命令、退出码、最小复现步骤、预期行为和经过脱敏的实际输出。

## 不要公开发布敏感内容

不得在 Issue、截图、日志、附件或复现仓库中提交真实、保密、未授权剧本，API Key、令牌、`.env` 内容，个人信息、内部路径、模型请求正文或其他敏感制作资料。复现只能使用合成、公版或已获得明确授权的最小样本。

安全漏洞和凭据泄露请依照 [安全政策](SECURITY.md) 使用 GitHub Private Vulnerability Reporting，不要创建包含细节的公开 Issue。

## 支持边界

本项目由维护者按可用时间维护，不提供响应时限、商业服务等级或个别项目的免费咨询。社区支持主要覆盖安装、CLI 行为、数据结构、可重复的软件缺陷和通用扩展方式。

以下事项不属于社区支持承诺：

- 代为上传、阅读或评审未公开剧本；
- 为具体项目判断主题、人物弧光、拍摄可行性或制作预算；
- 提供法律、版权、保险、职业安全或现实拍摄许可意见；
- 处理 DeepSeek 账户、余额、计费、服务可用性或供应商隐私政策；
- 为第三方修改版、过期版本或未记录环境提供长期兼容支持。

涉及 DeepSeek 服务本身的问题，请同时查阅并联系其官方支持渠道。AI 输出和本项目的封版状态不能替代真人专家、法务、制片部门及现实安全责任人的批准。

## 商业与定制需求

仓库暂不承诺商业支持、定制开发或项目级数据处理服务。可以创建一个不含保密信息的功能建议 Issue，说明通用需求；是否实现由维护者根据项目范围决定。
